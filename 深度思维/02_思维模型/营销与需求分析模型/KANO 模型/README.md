# KANO 模型

**KANO 模型**是由日本东京理工大学教授**狩野纪昭（Noriaki Kano）**于 1980 年代提出的一种用于**分类和优先排序用户需求**的定性分析模型。它通过分析**功能具备程度**与**用户满意度**之间的非线性关系，帮助产品团队科学地决定功能开发的优先级。

---

## 1. 核心维度

KANO 模型通过建立一个二维坐标系来分析需求：

- **横轴（具备程度）**：指产品提供某项功能或服务的程度（从“完全没有”到“完全具备”）。
- **纵轴（满意程度）**：指用户对该功能或服务的心理感受（从“非常不满意”到“非常满意”）。

## 2. 五类需求属性

根据这两个维度的交互关系，用户需求被划分为五种截然不同的类型：

### 🟢 必备/基本型需求 (Must-be / Basic Quality)

> **关键词：痛点、门槛、理所当然**

- **定义**：这是产品的基本功能。如果**没有**，用户会**极度不满意**；如果**有了**，用户认为是**理所当然**，并不会因此感到满意。
- **策略**：**必须做到 100% 满足**。这是进入市场的入场券，做不到就别发布。
- **案例**：手机能打电话、微信能发消息、酒店房间有床。

### 🔵 期望/一元型需求 (One-dimensional / Performance Quality)

> **关键词：痒点、线性相关、竞争力**

- **定义**：用户满意度与功能具备程度呈**线性正相关**。提供的越多、越好，用户越满意；反之则越不满意。
- **策略**：**竞争的主战场**。需要“人无我有，人有我优”，持续投入资源进行优化。
- **案例**：手机电池续航时间、汽车油耗、互联网产品的响应速度。

### 🔴 魅力/兴奋型需求 (Attractive / Excitement Quality)

> **关键词：爽点、惊喜、差异化**

- **定义**：用户意想不到的需求。如果**没有**，用户**无所谓**（因为没想到）；如果**有了**，用户会**非常惊喜**，满意度急剧上升。
- **策略**：**低成本高收益的突破口**。用于打造差异化，建立品牌口碑。
- **案例**：微信早期的“摇一摇”、火锅店提供的免费美甲/擦鞋服务、iPhone 初代的触控交互。

### ⚪ 无差异型需求 (Indifferent Quality)

> **关键词：无感、鸡肋**

- **定义**：用户根本不在意该功能，无论有没有，满意度都不受影响。
- **策略**：**避免投入**。属于资源浪费，应尽量减少或剔除。

### ⚫ 反向型需求 (Reverse Quality)

> **关键词：反感、负作用**

- **定义**：做得越多，用户越反感，满意度反而下降。
- **策略**：**必须剔除**。通常发生在该功能阻碍了用户原本的目标时。
- **案例**：过多的弹窗广告、繁琐的强制引导流程、复杂的安装捆绑包。

---

## 3. 模型启示与应用

### 优先级排序原则

在资源有限的情况下，通常遵循以下开发顺序：

1.  **必备型需求**（先活下来，不被骂）
2.  **期望型需求**（再活得好，有竞争力）
3.  **魅力型需求**（最后求发展，制造惊喜）

### 需求的生命周期演变

需求属性并非一成不变，而是随着时间推移和市场成熟而**降级**：

> **魅力型 ➔ 期望型 ➔ 必备型**

- _例子_：手机指纹解锁在刚推出时是“魅力型”；后来变成各家都有的“期望型”；现在如果一台手机没有生物识别，就是连“必备型”都没满足。
- **启示**：企业必须持续创新，挖掘新的魅力型需求，否则会被市场淘汰。
